# Microservice Description
- A microservice that computes scalar rewards using enhanced ICD-10 codes evaluation and non-enhanced ICD-10 codes evaluation.
- It uses three metrics to compute a single scalar reward that would be sent to (would act as an input to) the Reinforcement Learning Block (successor microservice).
- The three metrics are as follows:
  1. Ground Truth code values for the corresponding doctor's note (gt_codes)
  2. Enhanced ICD-10 code values generated by Task LLM via optimised doctor's notes (enh_codes)
  3. Non-enhanced ICD-10 code values generated by Task LLM via original doctor's notes (org_codes)
- This microservice will also be able to access the stored tree structure for ICD - 10 codes.
- Using the above four inputs, the microservice will generate a scalar reward between [-1.0, 1.0] which will then be given as an input along with other metrics to the Reinforcement Learning Block.

# Scalar Reward Calculation
- To use gt_codes, enh_codes, org_codes and ICD-10 tree structure to evaluate the required scalar reward that lies between [-1.0, 1.0].
- The rules for the generation of reward is as follows:
  If distance_between(gt_codes, enh_codes) < distance_between(gt_codes, org_codes):
    reward <- (0, 1.0]
  Else if distance_between(gt_codes, enh_codes) > distance_between(gt_codes, org_codes):
    reward <- [-1.0, 0)
  Else:
    If distance_between(enh_codes, org_codes) == 0:
      reward <- 1.0
    Else:
      reward <- [-1.0, 0)
- The function distance_between() would implement a relevant distance metric best suited for ICD-10 Tree structure.
- The function should also take into amount the hierarchy of the Tree structure as well while implementing the distance metric.

# Input 
- ICD-10 Tree structure dataset,
- Ground Truth code corresponding to current Doctor's note (gt_codes)
- Enhanced ICD-10 code values generated by Task LLM via optimised doctor's notes (enh_codes)   
- Non-enhanced ICD-10 code values generated by Task LLM via original doctor's notes (org_codes)

# Output
- Scalar reward value lying between [-1.0, 1.0]

# Ordered flow of execution
  |
  |
  | (input: {ICD-10 Tree structure dataset, gt_codes, enh_codes, org_codes})
  |
  | -> the microservice fetches gt_codes from the directory ./gt_codes/
  | -> the microservice recieves enh_codes from the api endpoint where the Task LLM POSTed the enhanced evaluation results
  | -> the microservice will also recieve org_codes from the api endpoint where the Task LLM POSTed the normal evaluation results
  |
  | -> implement a relevant distance metric as a function name distance_between()
  |
  | -> using the function distance_between() calculate scalar reward as mentioned above in the subsection 'Scalar Reward Calculation'
  |
  | -> POST this scalar reward R in an appropriate HTTPS endpoint for the next microservice to access it (Reinforcement Learning block)
  |
  | (output: R (scalar reward))
  |
  |
